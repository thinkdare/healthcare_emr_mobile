What changes

  • Remove: first name, last name, phone, license number, specialization, facility dropdown, emergency access toggle — the backend doesn't accept these.
  • Keep: email field.
  • Add: staff type dropdown (values from API contract), clinical rank selector (expanded cards showing capability chips, loaded from GET /clinical-ranks), active facility banner.
  • API call: POST /staff/invite with { email, staff_type, clinical_rank_id } and the X-Tenant-ID header.
  • On success: snackbar "Invitation sent to jane@clinic.com", form resets.
  • Clinical ranks are loaded once on initState and shown as selectable cards — not a dropdown — so the capability chips are visible without extra taps.
