Two approaches for the org admin section

Same screens in both — the difference is density and richness of information

A

Standard — follow existing patterns exactly

Org Profile: read-only card view with an Edit button that navigates to a form screen. Same card layout as StaffProfileScreen.

Staff Management: plain list of staff cards (name, type, rank, active badge). Tap a card → bottom sheet with two dropdowns (staff type, clinical rank) and an active toggle.

Invite Staff: clean form — email, staff type dropdown, clinical rank dropdown (loaded from API). No extra fields.

✓ Fastest to build   ✓ Consistent with rest of app   ✗ No at-a-glance org overview

B

Enhanced — richer admin-tool feel

Org Profile: stats header card at the top showing org-level counts (facilities, staff, patients) from GET /organizations/{id}/stats, then editable fields below. One screen, inline edit mode via pencil icon.

Staff Management: filter chips at the top (All / Doctor / Nurse / Admin / etc. and Active / Inactive toggle). Staff cards show capability chips (Prescribe, Labs, Emergency). Tap → bottom sheet editor.

Invite Staff: same form, but clinical rank picker shows rank name + capability summary so admin can see what they're granting.

✓ More useful for admins managing large teams   ✓ Stats give immediate context   ✗ More API calls, more state to manage

My recommendation: B (Enhanced)
Admins who manage facilities and staff need situational awareness — a bare list gives them no context. The extra API call for org stats is one request cached in the provider. The filter chips on Staff Management will pay off immediately for facilities with more than ~10 staff members. The capability summary on the rank picker prevents the common mistake of granting the wrong rank.
