Implementation notes

  • API uncertainty flagged: GET /v1/staff/memberships is documented as "current user's memberships." For admin use we assume it returns all facility staff when called with X-Tenant-ID and an admin token. Confirm with backend before building; if not, a GET /tenants/{id}/staff endpoint may be needed.
  • Filter chips are client-side only — filter the already-loaded list, no new API calls.
  • Clinical ranks loaded once on screen open via GET /clinical-ranks, cached in the bottom sheet state.
  • Edit calls PUT /staff/memberships/{id} with staff_type, clinical_rank_id, is_active.
  • "Remove from Facility" calls DELETE /staff/memberships/{id} with a confirmation dialog first.
