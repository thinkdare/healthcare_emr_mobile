A

Full component swap

Navigation + tabs + inputs + buttons + action sheets + swipe-to-delete + grouped lists. Everything feels native. Most work (~8 tasks to implement).

B

Structural only

Navigation chrome + tabs + grouped list style only. Buttons and inputs stay close to current. Faster (~4 tasks). Noticeable on close inspection but most users won't care.

C

Pragmatic — high-impact only

Navigation + tabs + buttons + action sheets for destructive actions + grouped lists. Skip swipe-to-delete and input field styling. Best bang for effort (~6 tasks).
